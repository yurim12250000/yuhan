vcs는 "version control system"의 약자로, 소프트웨어 개발에서 코드와 파일의 버전을 관리하는 시스템
DVCS는 "distributed version control system"의 약자로, 이것은 중앙 서버가 아닌 분산된 저장소에서 코드를 관리하는 시스템이다.
VCS DVCS 차이
vcs는 중앙 서버에서 코드를 관리하는 반면 dvcs는 각 개발자가 로컬에서 작업하고 병합하는 분산된 방식을 채택.

.gitignore
사용법
파일 생성:
먼저 Git 저장소의 루트 디렉토리에 .gitignore 파일을 생성.
무시할 파일 또는 디렉토리 지정:
.gitignore 파일에 무시할 파일 또는 디렉토리의 경로를 지정합니다. 각 항목은 새로운 줄에 작성.
와일드카드 사용:
와일드카드(*)를 사용하여 여러 파일을 한 번에 지정할 수 있습니다. 예를 들어, *.log는 모든 확장자가 .log인 파일을 나타냄.
디렉토리 무시:
특정 디렉토리를 무시하려면 디렉토리 이름 뒤에 슬래시(/)를 추가합니다. 예를 들어, logs/는 logs 디렉토리를 무시.
경로 지정:
파일이나 디렉토리의 상대 경로 또는 절대 경로를 지정할 수 있습니다. 상대 경로는 저장소의 루트 디렉토리를 기준으로 지정됨.
패턴 사용:
와일드카드 외에도 패턴을 사용하여 여러 파일을 한 번에 지정할 수 있습니다. 예를 들어, node_modules/는 node_modules 디렉토리와 그 하위 모든 파일을 무시한다.
중요한 파일 무시 방지:
Git이 기본적으로 무시하는 파일도 있다. 이러한 파일을 무시하지 않도록 하려면 무시 패턴에 !를 사용하여 예외를 추가할 수 있음.

작성방식
.gitignore 파일은 텍스트 파일이며, Git 저장소의 루트 디렉토리에 위치해야함.
각 줄은 무시하려는 파일 또는 디렉토리의 경로 또는 패턴을 포함해야 한다.
내용

.gitignore 파일에는 무시할 파일의 경로 또는 패턴을 지정한다.
경로는 상대 경로 또는 절대 경로로 지정할 수 있음.
패턴은 와일드카드(*)를 사용하여 여러 파일을 대표할 수 있다.
디렉토리를 무시하려면 디렉토리 이름을 지정하고 슬래시(/)를 추가한다.
특정 확장자를 가진 파일을 무시하려면 .확장자와 같이 지정.
와일드카드를 사용하여 특정 패턴을 일괄적으로 무시할 수 있다.
